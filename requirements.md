# 需求文档

## 介绍

为经验丰富的软件架构师构建个人作品集网站，展示作为"AI时代一人军团"的能力，主要通过InterviewPass案例研究展示产品开发、架构设计和数据工程的三位一体能力。

## 术语表

- **作品集系统**: 完整的个人作品集网站
- **语言管理器**: 负责双语言切换的组件
- **导航系统**: 带有语言切换的全局导航组件
- **英雄区块**: 主页介绍区域
- **案例研究展示**: InterviewPass项目的交互式展示
- **管道模拟器**: 显示数据处理模拟的终端风格组件
- **架构图**: 系统架构的交互式可视化表示
- **AI小部件**: Vertex AI驱动的浮动聊天机器人界面组件
- **翻译服务**: 管理双语内容的系统
- **MCP服务器**: 通过Model Context Protocol暴露作品集数据的独立服务
- **SEO层**: 搜索引擎优化基础设施（Metadata、Sitemap、Robots、JSON-LD）
- **数据分析展示**: 使用合成数据展示数据分析方法论的页面

## 需求

### 需求1: 双语言支持

**用户故事:** 作为访问者，我希望能够用中英文两种语言查看作品集，以便我能用首选语言理解内容。

#### 验收标准

1. 当网站加载时，语言管理器应初始化默认语言设置
2. 当用户点击语言切换按钮时，作品集系统应将所有UI文本切换到选定语言
3. 翻译服务应为所有面向用户的文本提供结构化双语内容
4. 当语言改变时，作品集系统应在页面导航中保持语言偏好
5. 作品集系统应显示中英文文本或提供语言间的无缝切换

### 需求2: 全局导航系统

**用户故事:** 作为访问者，我希望在所有页面上都有一致的导航，以便我能轻松探索作品集的不同部分。

#### 验收标准

1. 导航系统应突出显示"NiagaraDataAnalyst"标志
2. 导航系统应提供到首页、InterviewPass案例、数据分析、架构和关于部分的链接（共5个导航项）
3. 导航系统应包含突出的"中/En"语言切换按钮
4. 当点击导航链接时，作品集系统应导航到相应页面
5. 导航系统应在所有页面上保持可见和可访问

### 需求3: 首页英雄展示

**用户故事:** 作为访问者，我希望立即了解架构师的能力和价值主张，以便我能快速评估他们的专业知识。

#### 验收标准

1. 英雄区块应显示描述架构师能力的双语标题和副标题
2. 英雄区块应以左侧文本和右侧视觉元素的方式呈现内容
3. 英雄区块应包含代表开发->数据->洞察循环的抽象"无限循环"图标
4. 英雄区块应使用带有渐变文本样式的暗色模式美学
5. 当首页加载时，英雄区块应是第一个可见内容

### 需求4: 能力展示

**用户故事:** 作为潜在客户或雇主，我希望看到技术能力的具体证据，以便我能评估架构师的专业知识。

#### 验收标准

1. 作品集系统应显示包含三个能力卡片的"证明"部分
2. 作品集系统应展示工程能力，突出显示100%无服务器架构与AWS Lambda和Kinesis视频流
3. 作品集系统应展示AI和视觉能力，突出显示Amazon Rekognition和LLM集成
4. 作品集系统应展示数据循环能力，突出显示Airbyte和仓库分析
5. 当显示能力卡片时，作品集系统应以视觉吸引的网格布局呈现它们

### 需求5: 交互式案例研究展示

**用户故事:** 作为技术评估者，我希望详细探索InterviewPass案例研究，以便我能理解架构师的问题解决方法和技术实现。

#### 验收标准

1. 案例研究展示应为InterviewPass案例研究提供专门页面
2. 管道模拟器应显示带有模拟日志的终端风格控制台
3. 管道模拟器应显示带有视频占位符和AI洞察的报告仪表板
4. 管道模拟器应包含使用Recharts显示置信度分数随时间变化的折线图
5. 案例研究展示应在统一界面中集成所有模拟组件

### 需求6: 交互式架构可视化

**用户故事:** 作为技术利益相关者，我希望交互式地探索系统架构，以便我能理解技术设计和组件关系。

#### 验收标准

1. 架构图应显示包含六个节点的交互式图表：用户客户端、Kinesis、Lambda、Rekognition、Airbyte和Snowflake
2. 当用户点击任何架构节点时，架构图应显示包含技术规格的详细面板
3. 架构图应显示组件间的动画数据流连接
4. 架构图应使用reactflow库实现交互功能
5. 架构图应集成到案例研究页面中

### 需求7: AI聊天机器人集成

**用户故事:** 作为访问者，我希望与AI助手交互以了解更多架构师的能力，以便我能获得个性化信息和工作匹配洞察。

#### 验收标准

1. AI小部件应显示为带有呼吸动画的浮动启动按钮
2. 当点击启动器时，AI小部件应扩展为由Vertex AI (gemini-3.0-flash) 驱动的真实聊天窗口
3. AI小部件应提供用于职位描述分析的工作匹配模式标签
4. AI小部件应在移动设备上响应并正常工作
5. AI小部件应从作品集的所有页面保持可访问
6. API Route应实现流式响应（generateContentStream）
7. 系统提示词应包含完整的作品集知识库（个人简介、技能、项目经验）

### 需求8: 技术框架实现

**用户故事:** 作为维护作品集的开发者，我希望系统使用现代、可维护的技术构建，以便它能轻松更新和扩展。

#### 验收标准

1. 作品集系统应使用Next.js 14与App Router架构构建
2. 作品集系统应使用Tailwind CSS进行所有样式实现
3. 作品集系统应在所有代码文件中包含中文注释和文档
4. 作品集系统应通过适当的布局和路由结构集成所有组件
5. 作品集系统应确保所有文本内容使用语言管理器进行双语支持

### 需求9: 内容管理和文档

**用户故事:** 作为作品集所有者，我希望所有代码都有中文文档，以便我能有效地维护和扩展系统。

#### 验收标准

1. 作品集系统应包含解释组件功能的中文注释
2. 作品集系统应包含配置和设置的中文文档
3. 作品集系统应以结构化、可维护的格式组织翻译内容
4. 作品集系统应在内容和表示逻辑之间提供清晰分离
5. 作品集系统应在适当的地方包含中文变量名和函数描述

### 需求10: SEO与搜索优化

**用户故事:** 作为网站所有者，我希望网站在搜索引擎中对AI、软件开发、数据分析、产品设计等关键词有良好排名，以便潜在雇主和客户能发现我。

#### 验收标准

1. 所有页面使用Next.js Metadata API导出结构化元数据
2. 自动生成sitemap.xml包含所有公开页面
3. robots.txt正确配置，允许搜索引擎爬取公开内容
4. 首页包含schema.org Person JSON-LD结构化数据
5. 所有页面包含Open Graph和Twitter Cards元数据
6. SEO关键词覆盖4大领域：AI、软件开发、数据分析、产品设计

### 需求11: MCP服务器

**用户故事:** 作为求职者，我希望AI助手能通过MCP协议获取我的作品集信息，以便向潜在雇主推销我的能力。

#### 验收标准

1. MCP服务器暴露作品集Resources（profile/skills/projects/experience/contact）
2. MCP服务器提供Tools（search_skills/match_job/get_project_detail）
3. MCP服务器可通过Claude Desktop等客户端连接
4. 作品集数据与网站翻译数据保持一致

### 需求12: AI理解展示

**用户故事:** 作为技术评估者，我希望看到候选人对AI技术的深度理解，以便评估其AI工程能力。

#### 验收标准

1. 案例研究页包含AI理解展示区块
2. 展示内容覆盖Prompt Engineering、RAG、Agent架构、MCP协议
3. 展示Vertex AI集成能力和实际应用
4. 所有内容支持双语

### 需求13: 数据分析方法论展示

**用户故事:** 作为技术评估者，我希望看到候选人的数据分析能力的具体证据和方法论。

#### 验收标准

1. 网站包含独立的数据分析展示页面 `/data-analysis`
2. 使用预生成的合成股票数据展示分析流程（无需实时API调用）
3. 展示技术指标计算和可视化（MA/EMA/RSI/MACD/布林带）
4. 展示统计建模能力（收益率分布、相关性分析、回归分析、假设检验）
5. 分析流程按4步方法论组织（数据探索→指标计算→统计建模→结论）
6. 所有内容支持双语
