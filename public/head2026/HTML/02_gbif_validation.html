Why Taxonomy Validation Matters

Hamilton's tree inventory was recorded by municipal field staff, meaning species names were entered manually and inconsistently. Before the 10/20/30 biodiversity analysis could be performed, every scientific name needed to be checked against an authoritative global taxonomy to ensure the correct genus and family were assigned.

The Global Biodiversity Information Facility (GBIF) backbone taxonomy — the world's most comprehensive open-access species database — was used as the reference. Each of the 299 unique species names in the inventory was queried through the GBIF API to obtain a validated genus, family, confidence score, and match type.

The first pass revealed 18,503 genus mismatches and 7,365 family mismatches. The majority were caused by non-taxonomic entries (genus-only identifiers like "Fraxinus spp.", cultivar names, and hybrid notations) rather than true naming errors.
🛠️

Two-Pass Cleaning Process

A two-stage validation approach was applied:

Pass 1 — Full species list queried against GBIF API 18,503 genus mismatches identified 7,365 family mismatches identified

Non-taxonomic entries (e.g., Fraxinus spp., Malus spp., cultivar names with apostrophes) were filtered out. These represent valid inventory entries — they indicate the field recorder knew the genus but not the exact species — but they cannot be matched to a single taxonomy record.

Pass 2 — Re-queried with cleaned names only Genus mismatches reduced to 564 Family mismatches reduced to 0
After the second pass: zero family mismatches remained. The 564 remaining genus mismatches are exclusively hybrid and cultivar names that GBIF maps to a parent species genus — a known, accepted discrepancy in botanical taxonomy.
📋

The final GBIF-validated dataset adds five new columns to each of the 275,156 tree records, enabling accurate family-level diversity analysis in Step 3.
